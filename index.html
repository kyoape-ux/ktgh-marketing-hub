系統總覽

「行銷整合分析中心」是光田綜合醫院行銷組的一站式數據管理平台，涵蓋：

  • Facebook 粉專追蹤 — 貼文觸及、互動、點擊等 KPI 分析
  • 活動/講座追蹤 — 報名、出席、滿意度、預算管理
  • 新聞媒體曝光 — 媒體報導記錄與觸及統計
  • Campaign / KOL — 行銷活動成果與 KOL 合作追蹤
所有資料與 Google Sheets 雙向同步，本機 localStorage 作為快取，確保離線時也能查看。

登入與權限

進入系統時會出現登入選單，分為三種角色：

角色密碼權限範圍
🔑 組長需輸入密碼最高權限：檢視、編輯、刪除、同步、系統設定
✏️ 組員需輸入密碼可編輯：檢視、新增、編輯、刪除內容
👁 唯讀免密碼僅瀏覽：只能查看資料，無法修改
登入狀態僅在目前分頁有效，關閉分頁或重新整理後需重新登入。右上角會顯示目前角色徽章。

總覽 Dashboard

進入系統後的首頁，提供整體 KPI 概覽：

  • 四大 KPI 卡片 — 累計總觸及、互動次數、發文總篇數、平均每篇觸及
  • 月份觸及趨勢 — 橫條圖顯示每月觸及量與貼文數
  • 貼文類型分佈 — 甜甜圈圖顯示相片 vs 影片比例
  • 亮點貼文 — 自動篩選觸及最高的前 3 篇

期間篩選

左上角下拉選單可依「全部期間 / 年度 / 季度 / 月份」篩選，所有卡片與圖表會即時更新。

匯入貼文數據

支援從 Facebook 後台匯出的 Excel (.xlsx) 或 CSV 檔案批量匯入。

操作步驟

  1. 點擊右上角「+ 匯入數據」按鈕
  2. 選擇 Facebook 匯出的 .xlsx 或 .csv 檔案
  3. 系統自動解析欄位並匯入
  4. 匯入完成後自動跳轉至貼文列表
匯入時會自動比對日期+標題避免重複。建議每月初從 FB 後台匯出上月數據。

貼文成效列表

以表格形式列出所有貼文，可進行搜尋、篩選、排序。

功能說明

  • 搜尋 — 輸入關鍵字即時過濾標題
  • 期間篩選 — 依年度/季度/月份篩選
  • 新增貼文 — 手動新增單篇貼文（需編輯權限）
  • 編輯 / 刪除 — 點擊列表中的操作按鈕（需編輯權限）

每篇貼文記錄：日期、標題、類型、內容分類、觸及、互動、點擊、分享等指標。

組合分析

交叉比對多維度數據，找出最佳內容策略：

  • 類型 × 分類 — 相片/影片在不同內容分類下的表現
  • 趨勢對比 — 不同時間段的觸及與互動變化

內容日曆

以月曆視圖呈現貼文發布時間，方便掌握發文節奏：

  • 每日格內顯示當天發布的貼文數量
  • 可用「← 上月 / 下月 →」按鈕切換月份
  • 點擊日期可查看該日貼文詳情

A/B 分析

對比兩篇貼文的成效數據，判斷哪種策略更有效：

  1. 從下拉選單選擇「貼文 A」和「貼文 B」
  2. 系統自動顯示觸及、互動、點擊等指標的對比
  3. 以視覺化長條圖呈現差異

貼文預測

根據歷史數據預測下一篇貼文的預期表現：

  • 選擇貼文類型與內容分類
  • 系統顯示預估觸及、互動率、最佳發布時段

SOP 建議模板

根據不同內容分類提供標準作業流程建議：

  • 每個 SOP 卡片包含：發文前準備、撰文要點、發布策略、成效追蹤
  • 可作為新進人員教育訓練參考

活動成效看板

活動/講座的綜合 KPI 儀表板：

  • 四大 KPI — 總場次、平均出席率、平均滿意度、累積 FB 觸及
  • 活動類型分布 — 各類型活動的場次與出席率
  • 月份趨勢 — 每月活動場次與觸及
  • 滿意度分布 — 下拉選單依星等歸類活動

期間篩選

頂部下拉選單可依年度/季度/月份篩選，所有數據即時更新。

活動/講座列表

以表格形式列出所有活動記錄：

  • 篩選 — 期間（年/季/月）、活動類型、院區（向上/大甲）
  • 排序 — 依日期由新到舊
  • 操作 — 點擊「編輯」修改活動資料、「刪除」移除記錄（需編輯權限）

每筆活動記錄包含：日期、名稱、類型、院區、報名人數、出席人數、FB 觸及、預算、支出、滿意度。

新增活動回報

活動結束後，由組員填寫活動成效數據（需編輯權限）：

  1. 點擊側邊欄「新增活動回報」或列表頁的「+ 新增活動」
  2. 填寫活動名稱、日期、類型、院區
  3. 填入報名人數、出席人數、FB 觸及數
  4. 填入預算、實際支出、滿意度評分（1-5 星）
  5. 點擊「儲存活動記錄」
系統會根據活動類型自動提供預估數據（報名人數、出席率、觸及預測）供參考。

新聞媒體曝光

記錄與管理所有媒體報導：

  • 媒體列表 — 查看所有媒體報導記錄
  • 新增媒體記錄 — 填寫媒體名稱、類型、報導標題、日期、觸及數、連結

媒體類型包含：電視、網路新聞、報紙、雜誌、廣播等。

門診表題材庫

管理每月門診表相關的行銷題材與跨傳達管道宣傳紀錄：

  • 門診表題材庫 — 新增/編輯每月門診題材、負責醫師、上線日期
  • 月度傳達矩陣 — 一眼看出題材在哪些傳達管道（FB / LINE / 官網 / 影片等）有曝光
  • 門診統計 — 產出 KPI 數據與成效報表，可一鍵輸出 A4 推廣報告
門診題材可直接關聯跨傳達管道標籤，方便在「各傳達管道分析」中整合所有管道成效。

官網 / SEO

串接 Google Analytics 4 與 Search Console 資料：

  • 官網儀表板 — 每日使用者、工作階段、頁面瀏覽、平均停留時間等
  • SEO 關鍵字追蹤 — 記錄關鍵字的曝光、點擊、CTR、排名變化
  • JSON / CSV 匯入 — 批次匯入 GA4 / Search Console 匯出資料
  • 一鍵產出報告 — 輸出含 KPI、趨勢、Top 頁面 / 關鍵字的 A4 報告

LINE@ 追蹤

記錄 LINE 官方帳號的訊息推播與好友互動：

  • LINE 儀表板 — 好友數、封鎖率、訊息觸及、點擊率
  • 貼文列表 — 每則推播的曝光、點擊、相關標籤
  • 批次匯入 — 從 LINE Official Account Manager 匯出 CSV 直接貼入
  • 一鍵報告 — 產出 LINE@ 成效 A4 報表

影片總覽 (YT / IG)

以「方案 A：雙向輕度連動」整合 YouTube 與 Instagram 影片成效：

  • 影片列表 — 只記錄 date / platform / title / views / engage / tags / external_url，影片主檔仍由影片管理系統維護
  • 手動新增 — 發片時同步貼到行銷中心，即可進入跨傳達管道統計
  • JSON / CSV 批次匯入 — 每月從影片管理系統匯出一次資料、整批貼上即可
    • JSON 格式範例：[{"date":"2026-03-15","platform":"YouTube","title":"...","views":1200,"engage":86,"tags":["LDCT"],"external_url":"https://..."}]
    • CSV 表頭：date,platform,title,views,engage,tags,external_url
  • 一鍵報表 — 產出 YT / IG 合併 A4 報表（含平台分布、Top 10 高觸及影片）
影片的 tags 欄位會自動寫入共用標籤字典，可在「各傳達管道分析」中與其他管道合併統計。

各傳達管道分析

以「主題標籤」為軸，整合 8 個傳達管道成效（FB / 活動 / 媒體 / 門診 / LINE / 官網 SEO / KOL / YT・IG）：

  • 標籤字典 — 新增主題標籤後，各管道即可自由勾選
  • 跨管道統計 — 選擇期間與標籤，看到該題目在每個管道的觸及、互動、明細列表
  • 一鍵產出推廣報告 — 輸出含 6+ KPI 卡、傳達管道分布長條圖、各管道明細的 A4 彙總報告
適合月底向院方簡報「LDCT 這個月做了哪些宣傳、成效如何」。

各管道一鍵產出報表

每個管道（粉專 / 活動 / 媒體 / 門診 / LINE / 官網 / KOL / YT・IG）頁面上方都提供「產生報表」按鈕：

  • 依當前篩選期間即時運算
  • A4 排版、含 KPI 卡 + 長條圖 + 明細表
  • 右上角「列印 / 存 PDF」即可輸出

匯出月報

一鍵產出精美的行銷成效月報，可列印或存為 PDF：

  1. 點擊右上角「匯出月報」按鈕
  2. 月報以新視窗開啟，包含八個分頁：
    • 粉專成效 — KPI、月份趨勢、類型分布、Top 5 貼文
    • 活動成效 — KPI、完整活動列表
    • 媒體成效 — KPI、媒體報導列表
    • 門診表 — 題材與傳達管道矩陣
    • LINE@ — 推播成效與貼文列表
    • 官網 / SEO — GA4 指標與關鍵字
    • KOL — 合作紀錄與成效
    • YT / IG — 影片列表、Top 觸及
  3. 期間篩選 — 頂部可依年度/季度/月份篩選報表範圍
  4. 點擊「列印 / 存 PDF」按鈕輸出
月報配色為穩重的深綠色系，適合列印與正式簡報使用。

Google Sheets 同步

系統透過 Google Apps Script (GAS) 與 Google Sheets 雲端資料庫雙向同步：

同步方式

  • 自動同步 — 每次載入頁面時自動從 Sheets 拉取最新資料
  • 手動同步 — 點擊右上角「同步」按鈕立即更新
  • 推送資料 — 在設定頁面可將本機資料推送回 Sheets

Google Sheets 按鈕

點擊右上角「Google Sheets」可直接開啟原始試算表查看或編輯原始資料。

設定管理（僅限組長）

側邊欄最下方的「設定 / Google Sheets」：

  • GAS URL 設定 — 設定 Google Apps Script Web App URL，啟用雲端同步
  • KPI 目標值 — 設定每月觸及目標、互動率目標等
  • 重新同步 — 從 Sheets 重新拉取資料
  • 推送至 Sheets — 將本機資料推送到 Google Sheets
  • 清除快取 — 清除本機 localStorage 資料並重新載入
⚠️ 設定功能僅限組長角色可操作。GAS URL 變更會影響整個團隊的資料同步。

如何設定 Apps Script？（點擊展開）
Step 1 — 建立 Google Spreadsheet，複製試算表 ID（網址中 /d/ 後的那段）
Step 2 — 開啟 擴充功能 → Apps Script，貼上 apps-script-backend.gs 的程式碼，並將 SPREADSHEET_ID 換成你的 ID
Step 3 — 執行 initSheets() 函式一次（建立工作表標頭）
Step 4 — 點選「部署 → 新增部署」→ 類型選「網路應用程式」→ 執行身分「我」→ 存取權限「所有人」→ 部署
Step 5 — 複製產生的 Web App URL，貼回上方欄位並儲存
